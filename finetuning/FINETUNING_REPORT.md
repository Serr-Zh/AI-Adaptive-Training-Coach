# Отчёт об экспериментах по fine-tuning

## 1. Формализация задачи

В рамках работы решалась задача fine-tuning языковой модели для домена фитнес-рекомендаций. Практическая постановка задачи — **Question Answering в генеративной форме**, то есть **abstractive question answering / supervised instruction fine-tuning**.

Модель получает на вход вопрос пользователя о тренировках, физической активности, восстановлении или прогрессе и должна сгенерировать полезный и безопасный ответ в стиле AI fitness coach.

Пример входа:

```text
User question:
what are the five functional strength movements?

Coach answer:
```

Пример целевого выхода:

```text
you primarily use five basic movement patterns in daily life hinge (or bend-and-lift), lunge (or single-leg), push, pull, and rotate (or twist). functional strength exercises mimic these movement patterns, including deadlifts, squats, push-ups, chin-ups, step-ups, torso twists, and more.
```

Fine-tuning выбран не для добавления новых фактических знаний, а для адаптации поведения модели к домену фитнес-рекомендаций: модель должна отвечать в формате тренера, давать практичные рекомендации и учитывать safety-инструкцию.

## 2. Набор данных

Использован датасет `its-myrto/fitness-question-answers`.

Источник: Hugging Face Datasets.

Исходная структура датасета:

| Поле | Назначение |
|---|---|
| `Question` | вопрос пользователя о фитнесе |
| `Answer` | целевой ответ |

Исходный датасет содержит 965 примеров. Так как отдельная validation-выборка в датасете не использовалась, данные были разделены вручную с фиксированным `seed=42`.

Итоговое разделение:

| Split | Количество примеров |
|---|---:|
| train | 868 |
| validation | 97 |

После подготовки каждая запись была преобразована в формат для supervised fine-tuning causal language model:

| Поле | Назначение |
|---|---|
| `prompt` | системная инструкция и вопрос пользователя |
| `completion` | эталонный ответ |
| `text` | объединённый текст `prompt + completion`, используемый при обучении |

Шаблон prompt:

```text
You are an AI fitness coach. Answer the user's fitness question clearly, safely, and practically. Do not provide medical diagnosis. If the question contains pain, injury, or medical symptoms, recommend consulting a qualified professional.

User question:
{Question}

Coach answer:
```

## 3. Ограничения

Обучение выполнялось в GPU-среде.

Использованная среда:

| Параметр | Значение |
|---|---|
| Python | 3.12.13 |
| PyTorch | 2.10.0+cu128 |
| GPU | Tesla T4 |
| GPU memory | 14.56 GB |
| Precision | fp16 |
| Метод | PEFT / LoRA |

Локальная машина не имеет NVIDIA GPU и CUDA, поэтому локально выполнялись подготовка данных, настройка конфигураций и оформление результатов. Обучение выполнялось в GPU-среде.

Full fine-tuning не использовался, так как он требует обновления всех параметров модели и предъявляет более высокие требования к памяти. Вместо этого использован PEFT-подход LoRA, при котором обучаются дополнительные low-rank параметры, а базовая модель остаётся замороженной.

## 4. Стратегия экспериментов

Выбрана стратегия **прогрессии**.

Порядок базовых экспериментов:

1. `Qwen/Qwen2.5-0.5B-Instruct`
2. `Qwen/Qwen2.5-1.5B-Instruct`
3. `HuggingFaceTB/SmolLM2-1.7B-Instruct`

Логика порядка:

- сначала обучается меньшая модель `Qwen2.5-0.5B-Instruct`, чтобы получить базовую точку сравнения;
- затем используется более крупная модель того же семейства `Qwen2.5-1.5B-Instruct`, чтобы проверить влияние размера модели;
- затем используется альтернативная compact instruct-модель `SmolLM2-1.7B-Instruct`, чтобы сравнить не только размер, но и семейство модели.

Дополнительно были проведены два эксперимента с гиперпараметрами на лучшей базовой модели `Qwen/Qwen2.5-1.5B-Instruct`:

1. `qwen_1_5b_lora_r16` — увеличение LoRA rank с `r=8` до `r=16`;
2. `qwen_1_5b_lora_lr1e4` — уменьшение learning rate с `2e-4` до `1e-4`.

## 5. Конфигурация обучения

Общая конфигурация базовых экспериментов:

| Параметр | Значение |
|---|---:|
| Метод | LoRA |
| `r` | 8 |
| `lora_alpha` | 16 |
| `lora_dropout` | 0.05 |
| `target_modules` | `q_proj`, `k_proj`, `v_proj`, `o_proj` |
| `learning_rate` | 2e-4 |
| `max_steps` | 25 |
| `per_device_train_batch_size` | 1 |
| `gradient_accumulation_steps` | 8 |
| Effective batch size | 8 |
| `max_seq_length` | 512 |
| `fp16` | true |
| `gradient_checkpointing` | true |

Для эксперимента `qwen_1_5b_lora_r16` изменены параметры:

| Параметр | Базовое значение | Новое значение |
|---|---:|---:|
| `r` | 8 | 16 |
| `lora_alpha` | 16 | 32 |

Для эксперимента `qwen_1_5b_lora_lr1e4` изменён параметр:

| Параметр | Базовое значение | Новое значение |
|---|---:|---:|
| `learning_rate` | 2e-4 | 1e-4 |

## 6. Метрики качества и цены

Для оценки качества использовались:

| Метрика | Интерпретация |
|---|---|
| `eval_loss` | значение функции потерь на validation-наборе; меньше — лучше |
| `perplexity` | экспонента от `eval_loss`; меньше — лучше |
| `train_loss` | значение функции потерь на train-наборе |

Для оценки цены использовались:

| Метрика | Интерпретация |
|---|---|
| `training_time_minutes` | время обучения эксперимента |
| `model_size_b` | приблизительный размер базовой модели в млрд параметров |
| GPU memory | доступная память GPU |
| adapter checkpoint | сохранённый LoRA-адаптер |

Для построения диаграммы цена/качество использован показатель:

```text
quality_score = 1 / perplexity
```

Чем выше `quality_score`, тем лучше качество относительно perplexity.

## 7. Результаты экспериментов

| Эксперимент | Модель | r | alpha | lr | train_loss | eval_loss | perplexity | Время, мин |
|---|---|---:|---:|---:|---:|---:|---:|---:|
| `qwen_0_5b_qlora` | Qwen2.5-0.5B-Instruct | 8 | 16 | 2e-4 | 2.0820 | 1.4154 | 4.1180 | 1.344 |
| `qwen_1_5b_qlora` | Qwen2.5-1.5B-Instruct | 8 | 16 | 2e-4 | 1.9103 | 1.3397 | 3.8179 | 1.630 |
| `smollm2_1_7b_qlora` | SmolLM2-1.7B-Instruct | 8 | 16 | 2e-4 | 2.1165 | 1.7938 | 6.0123 | 1.374 |
| `qwen_1_5b_lora_r16` | Qwen2.5-1.5B-Instruct | 16 | 32 | 2e-4 | 1.5604 | 1.0152 | 2.7599 | 1.648 |
| `qwen_1_5b_lora_lr1e4` | Qwen2.5-1.5B-Instruct | 8 | 16 | 1e-4 | 2.3445 | 2.0247 | 7.5738 | 1.597 |

Примечание: часть технических имён файлов содержит суффикс `qlora`, однако фактически использовалась LoRA без 4-bit quantization, что зафиксировано в метриках как `method=LoRA` и `use_4bit=false`.

## 8. Сравнение базовых моделей

Сравнение трёх базовых моделей при одинаковой LoRA-конфигурации `r=8`, `alpha=16`, `learning_rate=2e-4` показывает:

| Модель | eval_loss | perplexity | Время, мин |
|---|---:|---:|---:|
| Qwen2.5-0.5B-Instruct | 1.4154 | 4.1180 | 1.344 |
| Qwen2.5-1.5B-Instruct | 1.3397 | 3.8179 | 1.630 |
| SmolLM2-1.7B-Instruct | 1.7938 | 6.0123 | 1.374 |

Лучшее качество среди базовых моделей показала `Qwen2.5-1.5B-Instruct`: у неё минимальные `eval_loss=1.3397` и `perplexity=3.8179`.

`Qwen2.5-0.5B-Instruct` показала немного худшее качество, но является самой лёгкой моделью из выбранных. `SmolLM2-1.7B-Instruct` в данной конфигурации показала худшее значение validation loss и perplexity.

## 9. Анализ влияния гиперпараметров

Для анализа гиперпараметров использовалась модель `Qwen/Qwen2.5-1.5B-Instruct`, так как она показала лучший результат среди базовых моделей.

### 9.1. Влияние LoRA rank

Сравнение:

| Эксперимент | r | alpha | lr | train_loss | eval_loss | perplexity |
|---|---:|---:|---:|---:|---:|---:|
| `qwen_1_5b_qlora` | 8 | 16 | 2e-4 | 1.9103 | 1.3397 | 3.8179 |
| `qwen_1_5b_lora_r16` | 16 | 32 | 2e-4 | 1.5604 | 1.0152 | 2.7599 |

Увеличение rank с `r=8` до `r=16` и `lora_alpha` с 16 до 32 улучшило качество: `eval_loss` снизился с 1.3397 до 1.0152, а perplexity — с 3.8179 до 2.7599. Это означает, что более выразительный LoRA-адаптер лучше подстроился под домен фитнес-вопросов.

При этом время обучения изменилось незначительно: с 1.630 до 1.648 минуты. В рамках проведённых экспериментов увеличение rank дало лучший баланс качества и цены.

### 9.2. Влияние learning rate

Сравнение:

| Эксперимент | r | alpha | lr | train_loss | eval_loss | perplexity |
|---|---:|---:|---:|---:|---:|---:|
| `qwen_1_5b_qlora` | 8 | 16 | 2e-4 | 1.9103 | 1.3397 | 3.8179 |
| `qwen_1_5b_lora_lr1e4` | 8 | 16 | 1e-4 | 2.3445 | 2.0247 | 7.5738 |

Уменьшение learning rate с `2e-4` до `1e-4` ухудшило результат в рамках заданного числа шагов: `eval_loss` вырос с 1.3397 до 2.0247, а perplexity — с 3.8179 до 7.5738. Это указывает на более медленную адаптацию модели к данным при меньшем learning rate.

Финальная выбранная модель:


## 12. Вывод

В рамках работы был реализован полный пайплайн PEFT fine-tuning для задачи Question Answering в домене фитнес-рекомендаций. Были подготовлены данные, выполнено разделение на train и validation, настроены и запущены эксперименты для трёх базовых LLM, сохранены метрики и LoRA-адаптеры.

Стратегия прогрессии позволила сначала сравнить несколько базовых моделей, а затем провести дополнительный анализ гиперпараметров на лучшей из них. По результатам экспериментов финальной конфигурацией выбрана `Qwen/Qwen2.5-1.5B-Instruct` с LoRA `r=16`, `alpha=32`, `learning_rate=2e-4`.
