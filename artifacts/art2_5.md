## Выбор модели для AI-системы

### 1. Жесткие атрибуты и ограничения

Для системы **AI Adaptive Training Coach** были выделены следующие жесткие атрибуты:

- модель должна быть доступна через внешний API;
- модель должна поддерживать структурированный JSON-ответ, так как система возвращает строго заданный `CoachResponse`;
- модель должна иметь невысокую задержку, поскольку используется в интерактивном сценарии;
- стоимость должна быть приемлемой для регулярных прогонов проверочного набора и дальнейшего использования в MVP;
- интеграция должна быть простой для текущего Python-кода, уже построенного вокруг OpenAI-style API;
- для текущей версии проекта достаточно текстовой модели, без обязательной мультимодальности.

Из этих требований следуют рабочие ограничения:

- structured outputs — обязательны;
- доступ по API — обязателен;
- приоритет отдается моделям с хорошим балансом качество / цена / простота интеграции.

### 2. Сужение круга кандидатов по общедоступной информации

По официальной документации были оставлены четыре наиболее релевантных кандидата:

- **OpenAI GPT-5.4 mini** — основная mini-модель нового поколения, хорошо подходящая для structured output и текущей архитектуры проекта;
- **OpenAI GPT-5.4 nano** — более дешёвый и быстрый вариант, который подходит как бюджетный baseline;
- **Google Gemini 3 Flash Preview** — сильный внешний кандидат по скорости и стоимости;
- **Anthropic Claude Haiku 4.5** — быстрый кандидат с поддержкой structured outputs, но менее удобный для текущего стека.

### 3. Сравнительная таблица

| Модель | Structured output | Интеграция с текущим кодом | Роль для проекта | Краткая оценка |
|---|---|---|---|---|
| **GPT-5.4 mini** | Да | Очень высокая | Основной кандидат | Лучший баланс качества и простоты интеграции |
| **GPT-5.4 nano** | Да | Очень высокая | Бюджетный fallback | Дешевле, но слабее по качеству |
| **Gemini 3 Flash Preview** | Да | Средняя | Альтернативный поставщик | Сильный внешний кандидат по цене и скорости |
| **Claude Haiku 4.5** | Да | Средняя | Резервный вариант | Быстрый и надёжный, но менее удобен для текущего стека |

### 4. Краткое обоснование конечного выбора

Для первой версии системы целесообразно выбрать **OpenAI GPT-5.4 mini**.

Причины выбора:

- модель проходит по всем жестким атрибутам;
- structured outputs поддерживаются нативно, что особенно важно для строгой схемы `CoachResponse`;
- текущий проект уже использует OpenAI-compatible подход, поэтому интеграция будет самой простой;
- по сравнению с `GPT-5.4 nano` эта модель лучше подходит как основная, так как даёт более сильный баланс между качеством адаптации тренировок и стоимостью;
- по сравнению с внешними кандидатами вроде Gemini и Claude она требует меньше изменений в текущем проекте.

### 5. Резервные варианты

- **GPT-5.4 nano** стоит оставить как запасной бюджетный вариант для массовых прогонов и дешёвого baseline.
- **Gemini 3 Flash Preview** имеет смысл протестировать как альтернативного поставщика модели.
- **Claude Haiku 4.5** можно рассматривать как резервный вариант, если позже потребуется экосистема Anthropic.
