## Процесс итеративных улучшений системы

Для проверки работы системы был построен процесс автоматического прогона тестовых примеров через API приложения.  
В качестве проверочного набора данных используется файл `data/validation_cases.json`, содержащий набор JSON-запросов для разных сценариев работы системы: генерация стартового плана, адаптация тренировки и безопасный отказ.

Проверка выполняется в два этапа. Сначала запускается API приложения:

```bash
python run.py
```
После этого запускается скрипт автоматической проверки:
```bash
python scripts/run_eval.py --input data/validation_cases.json --output results/eval_results.csv --base-url http://127.0.0.1:8000 --endpoint /coach
```
Скрипт последовательно загружает все примеры из проверочного набора, отправляет каждый запрос в endpoint `/coach`, получает ответ от API и сохраняет результаты в итоговую таблицу `results/eval_results.csv`.

В результате формируется сводная таблица, в которой для каждого тестового примера сохраняются исходный запрос и результат работы системы. Это позволяет повторно запускать один и тот же набор кейсов после изменений в системе, сравнивать результаты между итерациями и использовать процесс для дальнейших улучшений.

По итогам проверки было обработано 10 тестовых кейсов, из которых 10 завершились успешно, без ошибок. Таким образом, процесс загрузки проверочного набора данных, автоматического получения результатов от API и формирования итоговой таблицы реализован корректно.
